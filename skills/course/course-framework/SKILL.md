# 课程框架技能

课程框架技能用于管理课程的元数据和目录结构。

## 课程结构

```
├── metadata.json           (课程元数据)
└── course/                 (课程内容存储根目录，不存在需自动创建)
    ├── toc.md              (课程目录文件)
    └── ...                 (章节内容文件，由 course-content 技能生成)
```

### `metadata.json` 格式约束

```json
{
    "title": "课程名称",
    "description": "课程描述",
    "difficulty": "入门|进阶|高级",
    "structure": "节|章-节|阶段-章-节",
    "isEnv": false,
    "goals": [
        "课程目标1",
        "课程目标2"
    ]
}
```

### `course/` 目录格式约束

1. 必须命名为 `course/`
2. 必须是目录，若不存在需自动创建；若存在但不是目录时，需自动删除并创建
3. 必须包含 `toc.md` 文件，若不存在需自动创建

### `toc.md` 格式约束

#### 结构与逻辑流约定

- **1层结构：节**（适用于体验或临时性课程）
  - 逻辑流：[环境准备/背景引入] → [核心知识/操作步骤] → [总结/练习]
  - 数量约束：总计 3-5 节

- **2层结构：章-节**（适用于教学类、入门类课程）
  - 逻辑流：[基础理论] → [核心语法/特性] → [进阶技巧] → [实战案例/项目总结]
  - 数量约束：4-6 章；每章 2-4 节

- **3层结构：阶段-章-节**（适用于专业课程、体系化课程）
  - 逻辑流：[原理剖析与准备] → [系统化基础] → [高阶实战与优化] → [综合项目交付]
  - 数量约束：3-4 阶段；每阶段 2-3 章；每章 2-3 节

#### 目录格式

1. 使用 `#` 表示目录层级，每增一个 `#` 表示增加一个子层级（最多支持 `###`）
2. 使用 `[目录标题](./文件链接)` 格式，文件链接为 course 目录的相对路径，`.md` 格式
3. 标题必须是纯粹的，不得使用含序号格式命名（如"第x章"、"第x节"）

#### toc.md 示例

节结构：
```markdown
# [节](./{id1}.md)
# [节](./{id2}.md)
# [节](./{id3}.md)
```

章-节结构：
```markdown
# [章](./{id1}.md)
## [节](./{id11}.md)
## [节](./{id12}.md)
# [章](./{id2}.md)
## [节](./{id21}.md)
## [节](./{id22}.md)
```

阶段-章-节结构：
```markdown
# [阶段](./{id1}.md)
## [章](./{id11}.md)
### [节](./{id111}.md)
### [节](./{id112}.md)
## [章](./{id12}.md)
### [节](./{id121}.md)
### [节](./{id122}.md)
```

## 工作流程

### 1. 课程元数据

- 生成或修改 `metadata.json`
- 包含课程名称、描述、难度、结构、目标等

### 2. 课程目录

- 结合课程元数据信息，生成或修改 `course/toc.md`
- 确保结构和数量约束符合用户选择的层级模式

## 严格约束

1. 若有明确的层级结构，使用对应的格式进行目录结构定义
2. 若有明确的数量约束，确保节数数量符合数量要求
3. 不得对课程章节内容进行生成，章节内容由 `course-content` 技能负责
