# 课程内容技能

课程内容技能负责生成各章节的内容，所有内容保存到 `course/toc.md` 中定义的章节路径下。

## 支持的内容类型

- 专业章节内容（markdown 格式）
- 知识点卡片（AI智能知识卡片格式）
- 选择题（单选/多选，```question 代码块格式）
- 编程题（```validate 代码块格式）

## 知识点卡片格式

知识点卡片使用 markdown blockquote 格式嵌入：

```markdown
> ###### 🤖 **AI智能知识卡片**
>
> ###### ✅ **知识点：{title}**
>
> ###### 👉 **点击链接进行查看**🔗 <{shareUrl}>
```

其中 `title` 和 `shareUrl` 来自 `knowledge_card` MCP 工具返回的数据。

## 工作流程

### 1. 确定章节范围

读取课程元数据(`metadata.json`)和课程目录结构(`course/toc.md`)，确定要生成内容的章节。

### 2. 生成知识点卡片

针对每个章节，调用 `knowledge_card` MCP 工具：
- 传入该章节的详细内容（章节标题、知识点详情、学习目标、实践要求）
- MCP 工具返回知识点卡片数据（包含 `title` 和 `shareUrl` 等字段）

### 3. 生成章节内容

结合课程元数据和知识点卡片，生成完整的、专业的章节内容：

#### 格式要求
- 所有内容使用 markdown 语法编辑
- 题干和描述性文字使用 `<p>` 标签包裹（在题目 JSON 中）
- 仅生成目录结构中最后层级的章节内容

#### 知识点卡片嵌入规则
- 知识点卡片必须紧跟在介绍该知识点的段落之后立即插入
- 采用"边生成内容边插入"的方式
- **严禁**将所有知识点卡片堆积到章节末尾或开头
- 一个章节可以嵌入多个知识点卡片
- 嵌入顺序应与章节内容的逻辑流一致

#### 题目嵌入规则
- 选择题包裹在 ```question ... ```{{active}} 代码块中
- 编程题包裹在 ```validate ... ```{{active}} 代码块中
- 题目 JSON 必须可被 JSON.parse() 成功解析
- 完整格式规范参见 [question-format.md](references/question-format.md)

### 4. 保存章节内容

- **必须严格按照 `course/toc.md` 中定义的章节路径和文件名保存**
- **路径提取规则**：必须从 toc.md 的链接语法 `[标题](./路径)` 中直接提取 `./路径` 部分
- **严禁**自行推断、修改、重命名或添加任何前缀/后缀
- 保存前必须核对目标路径与 toc.md 中的链接路径完全一致

## 严格约束

1. 仅生成目录结构中最后层级的章节内容
2. 路径一致性（最高优先级）：必须从 toc.md 中直接提取章节的文件链接作为保存路径
3. 知识点卡片必须紧跟在介绍该知识点的段落之后，严禁堆积到末尾
4. 题目格式必须符合 [question-format.md](references/question-format.md) 中的规范
5. 必须使用 markdown 语法编辑章节内容
