# Code Quality Check

## 触发场景

- "检查代码质量"
- "运行 lint"
- "代码健康度评估"
- "找出代码问题"

## 工作流程

1. 识别项目使用的语言和框架
2. 查找现有的 lint/typecheck 配置
3. 运行相应的检查命令
4. 汇总问题并按类型分类
5. 提供修复建议

## 检查项

### 代码风格
- 命名规范
- 缩进和格式化
- 注释质量

### 潜在问题
- 未使用的变量/导入
- 类型不一致
- 空指针风险

### 安全
- 硬编码密钥
- 输入验证缺失
- 依赖漏洞

## 输出

提供结构化的问题列表，包含：
- 文件位置
- 问题类型
- 严重程度
- 修复建议
